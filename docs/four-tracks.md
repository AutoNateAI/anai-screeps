# Four Tracks

These are the working tracks for turning Screeps practice into AutoNateAI learning material.

## Track 1: Screeps Player

Play the game directly and turn mistakes into lessons: economy collapse, idle spawns, bad paths, poor defense, expansion problems, and failed attacks.

## Track 2: Screeps Developer

Build the engineering workflow around the game: local server, Git, tests, deployment, private server operations, CPU, memory, performance, and bot architecture.

## Track 3: AI Collaborator

Capture the prompts, review loops, debugging conversations, and model collaboration patterns that help students learn how to build software with AI.

## Track 4: Tournament Organizer

Develop the league operations: submissions, version locking, scoring, rankings, replay review, brackets, spectator experience, and anti-cheat expectations.
