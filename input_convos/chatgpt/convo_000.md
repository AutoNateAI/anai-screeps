Good call for the M1 — this is actually the officially recommended path (the community wiki says Docker install is the recommended method), and linux/arm64 is explicitly listed as a supported architecture, so it runs natively, no Rosetta/emulation needed.

**1. Install Docker Desktop**
Download the **Apple Silicon** build from docker.com/products/docker-desktop, install it, open it once so the whale icon shows running in your menu bar.

**2. Make a project folder**

bash
mkdir -p ~/screeps-server && cd ~/screeps-server


**3. Create config.yml** in that folder:

yaml
mods:
  - screepsmod-auth
  - screepsmod-admin-utils

bots:
  simplebot: screepsbot-zeswarm

serverConfig:
  tickRate: 1000

(You can make this file with nano config.yml, paste that in, Ctrl+O then Enter to save, Ctrl+X to exit.)

**4. Run the container**

bash
docker run --restart=unless-stopped --name MyScreepsServer \
  -v ~/screeps-server:/screeps \
  -p 21025:21025 \
  screepers/screeps-launcher

First run pulls the arm64 image and downloads/builds the mods — give it a few minutes. You'll see log output stream in your terminal; leave this running (or add -d before --name next time to run it detached in the background).

**5. Confirm it's up**
In a browser: http://localhost:21025 — if you get a page instead of a connection error, the server's live.

**6. Point the Steam Screeps client at it**
Open Screeps, find the server switcher on the login screen, add localhost:21025 as a custom server. Since screepsmod-auth is loaded, you'll register a local username/password rather than logging in via Steam.

**7. Getting your first room**
Same honest flag as before — the exact spawn command depends on what shows up once you're actually in. Run through steps 1–6, then tell me what you see (either in the terminal logs or once you're logged into the client), and I'll give you the exact next command instead of guessing at it now.
